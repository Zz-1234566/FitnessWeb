# Design Context

## Target Audience
健身新手（fitness beginners）— 主要在电脑端使用，查阅动作要领、组数和注意事项。移动端需要响应式适配，但不能出现页面超出框架或组件过大过小的情况。

## Brand Personality
极简高级风（minimalist premium）— 干净、克制、专业感。三个关键词：**clean, confident, precise**。

## Aesthetic Direction
参考 MuscleWiki 的交互式肌肉图谱风格：以人体肌肉图为核心交互入口，点击肌群即展示对应动作卡片。信息密度适中，重点突出动作名称、器械、难度和训练参数。

### Tone
- 专业但不冷冰冰 — 像一个有品位的私人教练
- 信息清晰优先，装饰克制
- 用留白和层次感传达"高级"，而非用花哨效果

### Anti-references
- 不要花哨的渐变、霓虹色、glassmorphism
- 不要过度装饰的卡片（阴影、边框、图标堆叠）
- 不要 template 感（千篇一律的 hero + 三列卡片）

## Theme
浅色为主 + 深色模式支持。

### Light Mode
- 背景：接近纯白的暖灰 oklch(0.98 0.002 80)
- 文字：深灰 oklch(0.2 0.01 250)，非纯黑
- 主色：#1890ff（Ant Design 默认蓝，项目已确立）
- 辅助色：肌肉图用柔和的蓝绿/暖橙区分肌群

### Dark Mode
- 背景：深灰 oklch(0.15 0.01 250)
- 文字：浅灰 oklch(0.9 0.005 80)
- 主色保持 #1890ff，适当降低饱和度
- 卡片背景略亮于页面背景，营造层次

## Typography
- 当前使用 AlibabaSans — 作为中文正文字体保留
- 英文/数字可搭配 Space Grotesk 或同等几何无衬线体
- 标题用 600/700 weight，正文 400
- 模块化字号梯度，标题与正文至少 1.25 倍对比

## Layout Principles
- 桌面端为主（主要使用场景是电脑），移动端做响应式适配
- 移动端关键约束：不超出视口框架，组件尺寸不能过大或过小
- 卡片网格用 `repeat(auto-fill, minmax(280px, 1fr))`，桌面端限制最大宽度
- 留白大方，不要把所有信息塞进一个屏幕
- 肌肉图作为 hero 交互元素，占据视觉焦点

## Component Patterns
- Ant Design 5 组件库为基础，通过 theme token 定制
- 内联样式为主（项目当前风格），复杂组件可抽 CSS Module
- 动作卡片：封面图/视频 + 名称 + 器械标签 + 难度标签 + 训练参数

## Motion
- 页面切换和筛选用简单的 fade + slide
- 卡片 hover 用 subtle scale(1.02) + shadow 变化
- 不用 bounce/elastic
- 尊重 prefers-reduced-motion
